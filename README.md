Week 8 Journal Reflection — CS 210 Portfolio Project

CS 210 Portfolio Submission: Corner Grocer Item-Tracking Program
Summarize the project and what problem it was solving.
The Corner Grocer Item-Tracking Program was designed to help store management analyze daily grocery purchase records and see how often each item was sold. By reading data from a text file and counting each item’s frequency, the program gives the store clear insights about which products are most popular. This allows for smarter decisions about product placement and inventory management.
What did you do particularly well?
I’m proud of my use of object-oriented programming principles, especially encapsulation and abstraction in the GroceryTracker class. I paid close attention to code readability, using clear variable and function names, modular design, and in-line comments that explain reasoning behind important decisions. Exception handling and input validation were built in to make the program more reliable and user-friendly.
Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
If I were to improve the program, I’d optimize data processing for larger files and make the menu system even more intuitive. I’d also add more comprehensive error handling and input validation, which would further increase the program’s security and reliability. These enhancements would make the code more efficient and better equipped for real-world use.
Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
The most challenging part was implementing reliable file handling and exception management—making sure that missing or corrupted files wouldn’t crash the program, but would instead prompt a clear error message. I spent extra time testing, debugging, and reading C++ documentation and online forums. I also leaned heavily on version control with GitHub, which allowed me to experiment, track changes, and avoid losing progress.
What skills from this project will be particularly transferable to other projects or course work?
This project helped me develop skills that will transfer to any programming work: writing clear, maintainable, and well-documented code, following style guides, and using version control. I also improved my ability to break down problems using object-oriented design, which will help in more complex projects and team environments.
How did you make this program maintainable, readable, and adaptable?
I kept functions focused and well-separated, used descriptive naming throughout, and included comments that clarify not just what the code does, but why certain approaches were taken. This structure means that anyone—including my future self—can easily understand and adapt the program as requirements change.
